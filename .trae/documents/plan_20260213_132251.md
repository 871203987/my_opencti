## OpenCTI 项目文档生成计划（更新版）

### 需要分析的目录（排除 opencti-java）

1. **opencti-platform/** - 主平台
   - `opencti-graphql/` - 后端 GraphQL API（约 60+ 模块，300+ 文件）
   - `opencti-front/` - 前端 React 应用（约 500+ 文件）
   - `opencti-dev/` - 开发配置

2. **opencti-worker/** - Python Worker 模块 ⬅️ 新增
   - `src/` - Worker 核心代码（worker.py, listen_handler.py, push_handler.py 等）
   - Docker 配置

3. **client-python/** - Python 客户端
   - `pycti/` - 核心库（api、connector、entities、utils）
   - `docs/` - 文档
   - `examples/` - 示例代码
   - `tests/` - 测试

4. **docs/** - 项目文档站点
   - `administration/` - 管理文档
   - `deployment/` - 部署文档
   - `development/` - 开发文档
   - `reference/` - 参考文档
   - `usage/` - 使用文档

5. **.github/** - GitHub 配置
   - `ISSUE_TEMPLATE/` - Issue 模板
   - `workflows/` - CI/CD 工作流

6. **.devcontainer/** - 开发容器配置

7. **.circleci/** - CircleCI 配置

8. **根目录配置文件**

### 执行步骤

**阶段1：根目录和配置文件**
- 分析根目录配置文件
- 生成项目根目录 MODULE_OVERVIEW.md

**阶段2：opencti-platform/opencti-graphql**
- 分析 src/ 下所有模块（config、database、domain、graphql、http、manager、modules 等）
- 生成后端模块说明文档

**阶段3：opencti-platform/opencti-front**
- 分析 src/ 下所有组件
- 生成前端模块说明文档

**阶段4：opencti-worker** ⬅️ 新增
- 分析 Worker 核心代码
- 生成 Worker 模块说明文档

**阶段5：client-python**
- 分析 pycti/ 下所有模块
- 生成 Python 客户端说明文档

**阶段6：docs**
- 分析文档结构
- 生成文档站点说明

**阶段7：CI/CD 和开发配置**
- 分析 .github、.devcontainer、.circleci
- 生成 CI/CD 说明文档

### 输出文件
每个目录下生成 `MODULE_OVERVIEW.md`，包含：
- 目录结构说明
- 各文件/子目录功能描述
- 关键代码逻辑说明
- 模块间依赖关系
